# EconAgent: マクロ経済活動シミュレーションのための LLM エージェント — Li et al. (2024)

Li et al. (2024)「EconAgent: Large Language Model-Empowered Agents for Simulating Macroeconomic Activities」(*ACL 2024*, 15523–15536; arXiv:2310.10436) の，LLM 駆動マクロ経済エージェントベースモデルの再現実装である．LLM 駆動の家計集団が毎月，**労働傾向** `p^w_i` (働く意欲) と **消費傾向** `p^c_i` (資産のうち消費する割合) の 2 つを，プロフィール・現在のマクロ変数・記憶を統合した知覚プロンプトに基づいて決定する．これらの個体決定が，労働/財市場 (生産・需要・賃金/物価調整)・累進課税 + 均等再分配・Taylor ルールの利子率政策を通じて波及する．論文の核心的知見は，事前定義ルールや訓練済みネットワークを用いず，LLM の意味的意思決定のみから，インフレ・失業・**Phillips 曲線**・**Okun の法則** といった古典的マクロ経済規則性が **ボトムアップに創発する** ことである．決定論的な [socsim](https://github.com/akitenkrad/rs-social-simulation-tools) コアが家計初期化・就業 Bernoulli ドロー・市場調整一様乱数・スケジューリング・指標を担い，非決定的な LLM レイヤは意思決定/リフレクションのメカニズムに閉じ込め，`socsim-llm` クレット (プロンプト→応答キャッシュ + `temperature=0` + 固定 seed) で擬似決定論化する．

## 二層の決定論 (最初に読むこと)

LLM 出力は socsim の bit 再現性の **外側** にある．したがって設計を 2 層に分ける:

- **決定論的 socsim コア** — 家計初期化 (パレート賃金・年齢分布)・就業 Bernoulli ドロー・市場調整一様乱数 (`ctx.rng`, ChaCha20)・スケジューリング・財政/金融会計・全指標．seed を固定すれば bit 単位で再現する．
- **非決定的 LLM レイヤ** — 毎月の労働/消費決定と四半期リフレクション．`socsim-llm` の `CachingClient` (`hash(prompt+model)` → 応答キャッシュ)・`temperature=0`・固定 seed で擬似決定論化する．プロバイダ順序は `socsim-llm` の `FallbackClient` による «Ollama 第一 → OpenAI フォールバック»．

再現性の本体はモデルではなく **キャッシュ** である．ウォームキャッシュは同一応答を再生するため，再実行は無料かつ安定する (論文は 1 シミュレーション約 30 ドル・約 2 時間と記す)．各実行は `run_metadata.json` にモデル・endpoint・温度・seed・cache-hit 率を記録する．ローカル既定モデル (`llama3.2`) は論文の `gpt-3.5-turbo` と異なるため，再現目標は **定性的** とする: Phillips 相関が負 (`r < 0`)・Okun 相関が負 (`r < 0`)・インフレ/失業が妥当な範囲，であって厳密な `r = −0.619` / `r = −0.918` ではない．

> 本プロジェクトは LLM レイヤを `socsim-llm` クレットに標準化しており，`reqwest` は **使わない** (HTTP は socsim-llm が所有する)．相互作用は市場仲介なので空間格子・ネットワーク (`socsim-grid` / `socsim-net`) は不要であり，依存は `socsim-core` + `socsim-engine` + `socsim-llm` のみである．

## インストールとクイックスタート

```bash
# Rust シミュレーションをビルド (socsim と socsim-llm の Ollama+OpenAI バックエンドを取得)
cargo build --release

# ローカル Ollama を起動しモデルを pull しておく．例:
#   ollama pull llama3.2:latest
export OLLAMA_HOST=http://localhost:11434
export OLLAMA_MODEL=llama3.2:latest
# 任意の OpenAI フォールバック:
#   export OPENAI_API_KEY=sk-...   OPENAI_MODEL=gpt-4o-mini

# 小規模実行 (スモーク; 論文は N=100, months=240)
cargo run --release -- run --n-agents 4 --months 12 --seed 42

# Python 可視化ツールを導入 (ワークスペース直下で)
uv sync

# 直近実行の可視化 (マクロ指標時系列 + Phillips/Okun 散布図 + Pearson r/p)
uv run econagent-tools visualize

# 実行設定と LLM メタデータの確認
uv run econagent-tools show-experiment-settings --results-dir results/latest
```

### オフライン (LLM 不要) スモーク

月次ループ全体・出力ライタ・Python 可視化は，スクリプト化した mock クライアントによりライブ LLM なしで検証できる:

```bash
cargo run --release --example mock_smoke -- results
uv run econagent-tools visualize
```

## スコープ

本リポジトリは `EconWorld` + 6-phase ループ上の 5 メカニズム・Ollama→OpenAI フォールバック + キャッシュの LLM 意思決定層・マクロ指標を提供する．Rust の 3 サブコマンドで駆動する: `run` (単一設定)・`sweep` (エージェント数 × 税率スケール × LLM モデルの感度分析)・`reproduce` (headline シナリオを一括実行し，Phillips 曲線と Okun の法則の相関を期待符号アンカーと突合する)．Python `econagent-tools` (`visualize` / `visualize-sweep` / `show-experiment-settings` / `reproduce`) がマクロ時系列と Phillips/Okun 散布図を scipy Pearson `r`/`p` 併記で描画する．`reproduce` とオフライン mock 意思決定クライアントにより，headline 知見は実 LLM なしで検証できる．COVID-19 外的介入プロンプトと賃金十分位別の不平等 (Gini) 分解は拡張点として残している．
